## How Treasury Share Management Works (ERC20-like)

The TreasuryDAO contract uses an **ERC20-style share system** for ownership tracking:

### Share Variables
```solidity
mapping(address => uint256) public shares;  // Each officer's share count
address[] public officers;                  // List of all officers
uint256 public totalShares;                 // Total shares issued
```

### Initial Setup
```solidity
initializeShares(
  [0x5a79...a359, 0x43d1...dc37, 0x3c4b...d8b4],  // Officer addresses
  [50, 30, 20]                                     // Share amounts
);
// Sets totalShares = 100
```

### Dilution Mechanism
When new shares are issued (dilution event):
```solidity
// Add new officer with 11 shares
updateShares(0xNEW_INVESTOR, 11);

// Now:
// - Michael: 50 shares (was 50%, now 50/111 = 45.05%)
// - Richard: 30 shares (was 30%, now 30/111 = 27.03%)
// - Cathie: 20 shares (was 20%, now 20/111 = 18.02%)
// - New Investor: 11 shares (was 0%, now 11/111 = 9.91%)
// - totalShares = 111 (increased from 100)
```

### How Percentages Are Calculated
```solidity
function getSharePercentage(address officer) public view returns (uint256) {
    if (totalShares == 0) return 0;
    return (shares[officer] * 10000) / totalShares;  // Returns basis points (6000 = 60.00%)
}
```

### Profit Distribution Formula
```solidity
function distributeProfits(uint256 totalAmount, bytes32 documentHash) external {
    for (uint256 i = 0; i < officers.length; i++) {
        address officer = officers[i];
        uint256 officerShares = shares[officer];

        // Calculate proportional payout
        uint256 payout = (officerShares * totalAmount) / totalShares;

        // Transfer USDC to officer
        usdc.transfer(officer, payout);
    }
}
```

### Key Features
1. **No ERC20 Token**: Uses internal mapping, not a separate ERC20 contract
2. **Dilution Support**: Can issue new shares, which dilutes existing holders
3. **Permissionless**: Anyone can call `updateShares()` to modify allocations
4. **Automatic Distribution**: Profit payments calculated automatically based on current share ratios
5. **Dynamic**: Total shares can increase/decrease as shareholders join/leave
